# 04. Прикладная система — нативное приложение

## 1. Архитектура

### 1.1 Выбор технологии

Требования к UI:
- Работа с фото, видео и webcam
- Отзывчивый интерфейс (инференс не блокирует окно)
- Профессиональный вид для защиты
- Кроссплатформенность

Рассмотренные варианты:

| Технология | Pro | Contra | Итог |
|---|---|---|---|
| Gradio (web) | Быстрый прототип | Требует браузер, не нативный | ❌ |
| cv2.imshow | Минимум кода | Примитивный UI, нет кнопок | ❌ |
| tkinter | Встроен | Устаревший вид | ❌ |
| **PySide6 (Qt)** | **Профессиональный UI, QThread, кросс-ОС** | **Зависимость 300MB** | ✅ |

### 1.2 Общая схема

```
┌────────────────────────────────────────────────────────┐
│              MainWindow (PySide6)                      │
│  ┌──────────┬─────────────────────┬──────────────┐    │
│  │ Controls │   Video Display     │ Statistics   │    │
│  │  Model   │                     │  counts/     │    │
│  │  Mode    │   QLabel + pixmap   │  defect %    │    │
│  │  Conf    │   (60 FPS update)   │  log         │    │
│  │  Start   │                     │              │    │
│  └──────────┴─────────────────────┴──────────────┘    │
├────────────────────────────────────────────────────────┤
│  InferenceWorker (QThread) ─ фоновый поток            │
│    → model.track(frame, ...)                          │
│    → фильтры, сглаживание, inertia                    │
│    → emit frame_ready (Qt Signal)                      │
└────────────────────────────────────────────────────────┘
```

Главное окно и рабочий поток обмениваются через Qt-сигналы — UI никогда не блокируется.

## 2. Режимы работы

| Режим | Источник | Обработчик |
|---|---|---|
| 📷 Фото | JPG/PNG | `_infer_single()` — разовый прогон `model.predict` |

## 3. Проблемы стабильности и их решение

На реальных видео обнаружены три класса проблем. Для каждой реализовано решение:

### 3.1 Проблема — рамка на весь экран

**Симптом:** модель иногда выдаёт bbox, покрывающий 80-100% кадра.

**Причина:** при нечёткой картинке (motion blur, плохой свет) модель "угадывает" что бутылка занимает весь кадр.

**Решение — area filter:**
```python
MAX_BBOX_AREA_RATIO = 0.75  # > 75% кадра — выбрасываем
MIN_BBOX_AREA_RATIO = 0.002 # < 0.2% — тоже
```

### 3.2 Проблема — моргание детекции между кадрами

**Симптом:** на 1-2 кадрах модель теряет объект → bbox исчезает → появляется снова.

**Причина:** естественная вариативность confidence близко к порогу.

**Решения:**
1. **ByteTrack** через `model.track(..., persist=True, tracker="bytetrack.yaml")`
   Трекер связывает детекции между кадрами через Kalman-фильтр и назначает стабильные `track_id`.

2. **EMA-сглаживание координат bbox:**
```python
# При каждой новой детекции:
new_xyxy = alpha * previous_xyxy + (1 - alpha) * current_xyxy
# alpha=0.5 для сильных, alpha=0.8 для слабых классов
```

3. **Anti-flicker буфер:** если трек не обновился в этом кадре, показываем его ещё до HOLD_FRAMES_STRONG=3 кадров (для сильных классов) или HOLD_FRAMES_WEAK=10 (для слабых).

### 3.3 Проблема — слабые классы (missing_label) не обнаруживаются

**Симптом:** класс `missing_label` редко появляется, хотя визуально есть.

**Причина:** модель на этом классе имеет max conf ~0.40 (см. [per-class метрики](03_detection_final.md#5-результаты)), тогда как у других 0.70-0.88.

**Решение — per-class conf thresholds:**
```python
PER_CLASS_CONF = {
    "deformed_bottl":   0.45,
    "misaligned_label": 0.45,
    "missing_cap":      0.45,
    "normal":           0.45,
    "missing_label":    0.25,  # ← слабый класс, порог ниже
    "neck_deformed":    0.25,
}
```

Плюс **MIN_HITS=2** для слабых классов — защита от одиночных false-positive. Трек попадает в счётчик только после подтверждения минимум на 2 кадрах.

## 4. Итоговый набор стабилизаций

| Механизм | Параметр | Что делает |
|---|---|---|
| Area filter | 0.002 ≤ ratio ≤ 0.75 | Откидывает слишком большие и мелкие боксы |
| Per-class conf | 0.25 / 0.45 | Слабым классам — ниже порог |
| ByteTrack | persist=True, IoU=0.5 | Трекинг между кадрами, стабильные ID |
| EMA сглаживание | α=0.5 / 0.8 | Плавное движение bbox по координатам |
| Anti-flicker hold | 3 / 10 кадров | Не теряем бокс при пропусках |
| MIN_HITS | 2 для слабых | Подтверждение перед счётом |
| Dedup by track_id | — | Одна бутылка = один инкремент |

## 5. Интерфейс

### 5.1 Layout

Окно разделено на 3 вертикальные зоны через `QSplitter`:

**Левая панель (220px):**
- Выбор модели из dropdown (все `runs_det/*/weights/best.pt`)
- Выбор режима (radio buttons): Фото / Видео / Live
- Слайдер «Порог уверенности»
- Кнопки: Открыть файл, ▶ Старт, ■ Стоп, ↺ Сброс, 💾 Сохранить лог

**Центр:**
- Большой `QLabel` с видеопотоком (автоматический resize)
- Внизу: большой цветной badge с текущим классом и % уверенности
- Справа снизу: текущий FPS

**Правая панель (240px):**
- По каждому классу: прогресс-бар с количеством
- Общие счётчики: кадров, обнаружено, дефектов
- Доля дефектов в %
- Лог консоль

### 5.2 Цветовая схема

Тёмная тема (Catppuccin-inspired) — легче для глаз на долгих сессиях инспекции.

Цветовое кодирование классов:
- 🟢 зелёный — `normal` (годная)
- 🔴 красный — `deformed_bottl`, `neck_deformed` (сильные дефекты)
- 🟠 оранжевый — `missing_cap`, `misaligned_label`
- 🟡 жёлтый — `missing_label`

Рамка вокруг bbox окрашена по типу дефекта — оператор мгновенно видит, что не так.

## 6. Технические детали реализации

### 6.1 Threading

```python
class InferenceWorker(QThread):
    frame_ready = Signal(np.ndarray, str, float, float)   # annotated, cls, conf, fps
    stats_updated = Signal(dict)

    def run(self):
        model = YOLO(self.model_path)
        cap = cv2.VideoCapture(self.source)
        while not self._stop:
            ok, frame = cap.read()
            # ... inference + annotation
            self.frame_ready.emit(annotated, cls_name, conf, fps)
```

QThread не блокирует UI. Остановка через `_stop` flag + `QMutexLocker`.

### 6.2 Рендеринг кадров

```python
def _display_frame(self, frame):
    rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    qi = QImage(rgb.data, w, h, ch * w, QImage.Format_RGB888)
    pixmap = QPixmap.fromImage(qi).scaled(label_w, label_h,
                                          Qt.KeepAspectRatio,
                                          Qt.SmoothTransformation)
    self.video_label.setPixmap(pixmap)
```

Скорость рендеринга не узкое место — даже при 60 FPS нагрузка UI минимальна.

### 6.3 CSV-логирование

Каждая детекция пишется в локальный `_log_rows` list. По нажатию 💾 — сохранение в CSV через QFileDialog.

Поля: `timestamp, class, confidence, fps`.

## 7. Производительность

### 7.1 Live-режим на Mac (Apple Silicon CPU)

- Входное разрешение: 1280×720 (webcam)
- Инференс imgsz=640
- Средний FPS: **30-35** — больше чем реальный FPS камеры (25-30)
- UI отзывчив: < 16 мс reaction time

### 7.2 Обработка видеофайла

- 1080p видео, 932 кадра
- Время обработки: 29.4 сек (**31.7 FPS**)
- Быстрее чем реальное время (30 FPS)

### 7.3 Использование памяти

- Idle: ~400 MB (PySide6 + Python + модель)
- Running: ~500-600 MB
- Модель в памяти: ~25 MB (включая torch tensors)
