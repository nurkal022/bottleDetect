# Отчёт по дипломной работе
## Лёгкая система компьютерного зрения для контроля дефектов на конвейере

**Автор:** Нурлыхан
**Тип работы:** Бакалаврская ВКР
**Задача:** Object Detection (локализация дефектов bbox-ами)
**Дата:** 2026-04-19

---

## 1. Введение

### 1.1 Актуальность

Контроль качества на производственных линиях — одна из ключевых задач промышленной автоматизации. Ручная инспекция бутылок на конвейере отличается высокой трудоёмкостью, человеческим фактором и невозможностью обработки больших скоростей потока. Системы машинного зрения позволяют решить эту задачу автоматически, работая в реальном времени.

В отличие от классификации (вся картинка → один класс), **детекция** выделяет каждый объект отдельным bounding box-ом. Это даёт:
- Возможность работать с несколькими бутылками в одном кадре
- Точную локализацию дефекта на корпусе
- Независимость от кадрирования камеры

Важное ограничение встроенных систем — вычислительная мощность. Поэтому в работе исследуется **лёгкая (nano)** модель YOLOv8, которая сочетает приемлемую точность с высокой скоростью на CPU без GPU.

### 1.2 Цель

Разработать и обучить лёгкую нейросетевую систему детекции дефектов пластиковых бутылок, способную работать в реальном времени на обычном CPU, и реализовать прикладной прототип с нативным GUI.

### 1.3 Задачи

1. Подготовить и разметить датасет дефектов бутылок в формате YOLO
2. Обучить YOLOv8n detection
3. Получить метрики качества (mAP, Precision, Recall) и скорости (FPS, latency)
4. Разработать нативное десктопное приложение с поддержкой фото, видео и live-режима
5. Реализовать стабильное отслеживание объектов в реальном времени (ByteTrack)

## 3. Методология

### 3.1 Модель

**YOLOv8n (nano detection)** — pretrained на COCO, дообучение на bottle-defect.

| Параметр | Значение |
|---|---|
| Архитектура | YOLOv8 (C2f backbone + PAN neck + Detect head) |
| Слоёв | 130 |
| Параметров | 3,012,018 (3.01M) |
| FLOPs @ imgsz=640 | 8.1 G |
| Размер весов (best.pt) | 6.23 MB |

### 3.2 Гиперпараметры обучения

| Параметр | Значение |
|---|---|
| epochs | 100 |
| imgsz | 640 |
| batch | 32 |
| optimizer | AdamW |
| lr0 | 0.001 |
| patience (early stopping) | 20 |
| seed | 42 |
| device | NVIDIA RTX 5080 Laptop (16GB VRAM, CUDA 13) |

**Время обучения:** ~9 минут (100 эпох полностью, без early stopping).

### 3.3 Оборудование

- **Тренировка:** GPU NVIDIA RTX 5080 Laptop (16GB), PyTorch 2.9.1 + CUDA 13.0
- **Инференс (замеры):** CPU Apple Silicon (M-series) через локальное окружение
- **Развёртывание:** нативное приложение PySide6 на macOS

### 4.2 Метрики по классам

| Класс | Images | Inst. | Precision | Recall | mAP50 | mAP50-95 |
|---|---|---|---|---|---|---|
| deformed_bottl | 75 | 75 | 0.995 | **1.000** | **0.995** | 0.776 |

| normal | 73 | 93 | 0.943 | **1.000** | 0.986 | 0.766 |
| **все** | **204** | **265** | **0.961** | **0.953** | **0.984** | **0.622** |

**Наблюдения:**
- Лучший mAP50 (≥0.988) у 4 классов: `deformed_bottl`, `neck_deformed`, `missing_cap`, `misaligned_label`
- Худший класс — `missing_label` (mAP50 = 0.944). Recall 0.839 означает, что ~16% отсутствующих этикеток пропускаются
- `normal` имеет Recall 1.0 — ни одна «годная» бутылка не классифицирована как дефектная
- mAP50-95 ниже у `missing_cap` (0.451) и `missing_label` (0.446) — хорошая детекция при низком IoU-пороге, но bbox-ы не идеально точные при строгом пороге

## 5. Анализ

### 5.1 Качество

Модель показала mAP@0.5 = 0.984, что для задачи промышленной инспекции является **высоким результатом**. Класс `normal` имеет Recall = 1.0, то есть система никогда не "бракует" годную бутылку — это критично для минимизации ложных срабатываний в production.

Слабое место — класс `missing_label` (mAP50 = 0.944). Recall 0.839 означает, что каждая шестая бутылка с отсутствующей этикеткой пропускается. Причина — визуальное сходство с `misaligned_label` на некоторых ракурсах.

### 5.2 Скорость vs точность

При imgsz=512 вместо дефолтного 640 модель теряет минимум в точности (<1% mAP), но получает прирост в скорости в ~1.5×. Это оптимальный режим для live-детекции.

### 5.3 Сравнение с классификацией

В предыдущей итерации работы исследовалась классификация (вся картинка → один класс). Детекция даёт следующие преимущества:

| Аспект | Classification | Detection (наша) |
|---|---|---|
| Несколько бутылок в кадре | ❌ | ✅ |
| Локализация дефекта | ❌ | ✅ bbox |
| Практическое применение | Ограничено | Полное |
| Качество на bottle-defect | 100% (простой датасет) | mAP50 = 98.4% (сложнее) |
| Инференс на CPU | ~320 FPS | 33 FPS |

Детекция медленнее, но функционально полноценнее для реального конвейера.

## 6. Прикладная система

### 6.1 Нативное десктопное приложение

**Файл:** [app_native.py](../app_native.py)
**Технологии:** PySide6 (Qt для Python), OpenCV, Ultralytics

### 6.2 Режимы работы

| Режим | Вход | Что делает |
|---|---|---|
| 📷 Фото | JPG/PNG | Одиночная детекция с bbox и меткой |
| 🎬 Видео | MP4/AVI/MOV | Покадровая обработка |
| 📹 Live | Веб-камера | Real-time детекция |

### 6.3 Интерфейс

```
┌──────────────┬───────────────────────┬───────────────┐
│ Модель       │                       │ Счётчики      │
│ Режим        │    BBOX OVERLAY       │ deformed..    │
│ Conf slider  │    (видео/фото)       │ missing_cap   │
│ Управление   │                       │ missing_label │
│              │    [badge] [FPS]      │ neck_deformed │
│              │                       │ normal        │
│              │                       │ Дефектов: 23% │
└──────────────┴───────────────────────┴───────────────┘
```

### 6.4 Стабильность детекции

Реализованы три механизма стабильности:

1. **ByteTrack** — трекер присваивает каждому объекту уникальный ID и связывает детекции между кадрами. Исключает случайные пропуски в счётчиках.
2. **Anti-flicker буфер** — если модель пропустила объект на 1–3 кадрах, UI продолжает показывать предыдущий bbox. Устраняет «мигание» на живом видео.
3. **Дедупликация по track_id** — одна бутылка считается один раз за весь период присутствия в кадре, а не за каждый кадр.

### 6.5 Производительность на Mac

| Режим | Разрешение | FPS |
|---|---|---|
| Photo | до 4K | ~10–15 (одиночный прогон) |
| Video | 1080p | 30–35 FPS |
| Live | 720p (webcam) | 30+ FPS |

## 7. Выводы

1. **YOLOv8n detection** решает задачу обнаружения дефектов бутылок с mAP@0.5 = 0.984 и Recall = 0.953 на валидации. Для промышленной применимости это высокий результат.

2. **Лёгкая модель (3M параметров, 6 MB)** обеспечивает 33 FPS на обычном CPU — достаточно для конвейера со скоростью до 10 бутылок/сек с большим запасом.

3. **Класс `normal` имеет Recall = 1.0** — система не даёт ложных браков. Критично для минимизации потерь годной продукции.

4. **Худший класс — `missing_label`** (mAP50 = 0.944). При внедрении рекомендуется собрать ещё 100–200 изображений с отсутствующими этикетками и дообучить модель.

5. **Нативное приложение с ByteTrack** обеспечивает стабильную детекцию без мигания на live-видео и корректный подсчёт уникальных объектов.
